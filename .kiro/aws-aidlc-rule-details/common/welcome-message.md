# 👋 欢迎使用 AI-DLC（AI驱动的开发生命周期）！👋

我将指导您完成一个自适应软件开发工作流，该工作流会智能地根据您的具体需求进行调整。

## 什么是 AI-DLC？

AI-DLC 是一个结构化但灵活的软件开发过程，可以适应您项目的需求。可以把它想象成拥有一位经验丰富的软件架构师，他会：

- **分析您的需求**并在需要时提出澄清问题
- **基于复杂性和风险规划最优方法**
- **跳过不必要的步骤**以应对简单变更，同时为复杂项目提供全面覆盖
- **记录一切**，让您拥有决策和理由的完整记录
- **指导您完成每个阶段**，提供清晰的检查点和批准门

### 阶段分解：

**启动阶段** - *规划和应用程序设计*
- **目的**：确定构建什么以及为什么
- **活动**：理解需求，分析现有代码（如果有），规划方法
- **输出**：清晰的需求、执行计划、并行开发工作单元数量的决策
- **您的角色**：回答问题，审查计划，批准方向

**构建阶段** - *详细设计、实现和测试*
- **目的**：确定如何构建
- **活动**：详细设计（需要时），代码生成，全面测试
- **输出**：可工作的代码、测试、构建指令
- **您的角色**：审查设计，批准实现计划，验证结果

**运营阶段** - *部署和监控（未来）*
- **目的**：如何部署和运行
- **状态**：未来部署和监控工作流的占位符
- **当前状态**：构建和测试活动在构建阶段处理

## 关键原则：

- ⚡ **完全自适应**：每个阶段根据您的需求独立评估
- 🎯 **高效**：简单变更仅执行必要阶段
- 📋 **全面**：复杂变更获得完整处理和所有保障措施
- 🔍 **透明**：工作开始前您可以看到并批准执行计划
- 📝 **有记录**：所有决策和变更的完整审计跟踪
- 🎛️ **用户控制**：您可以请求包含或排除阶段

## 接下来会发生什么：

1. **我将分析您的工作区**以了解这是新项目还是现有项目
2. **我将收集需求**并在需要时提出澄清问题
3. **我将创建执行计划**显示我建议运行哪些阶段以及原因
4. **您将审查并批准**计划（或请求更改）
5. **我们将执行计划**在每个主要阶段设置检查点
6. **您将获得可工作的代码**以及完整的文档和测试

AI-DLC 过程适应：
- 📋 您的意图清晰度和复杂性
- 🔍 现有代码库状态
- 🎯 变更的范围和影响
- ⚡ 风险和质量要求

让我们开始吧！
